# Inference on a Single Image

Once `best_ctc.pt` is on disk, the entire inference pipeline is a
single command. This document covers:

1. How to run inference (with or without accuracy metrics).
2. What the script does, step by step.
3. How to read the FINAL RESULT banner.
4. The full Makefile command reference.

---

## 1. The simplest invocation

```bash
make ctc-infer
```

This:
- loads `model/best_ctc.pt`
- reads `samples/test.png`
- shows a 2-panel plot (image + per-frame argmax trace)
- prints the FINAL RESULT banner

---

## 2. With accuracy metrics

The script supports three ways to specify the ground truth:

| Method | Example | When to use |
|--------|---------|-------------|
| **Explicit flag** | `make ctc-infer GT=12345` | You know the ground truth, image filename doesn't include it |
| **Auto-extract from filename** | `make ctc-infer IMAGE=samples/sample_L7_1234567.png` | You're using samples generated by `make ctc-gen-one` (which embed the digits in the filename) |
| **Skip comparison** | `make ctc-infer NO_GT=1` | You're testing on a third-party image without a known label |

The auto-extract uses the regex `(\d+)$` on the filename stem, so
anything like `..._1234567.png` works.

### Example output

```
============================================================
  FINAL RESULT:  1234567
  (length: 7 digits)
------------------------------------------------------------
  Ground truth  :  1234567
  Sequence acc  :  OK ✔ (exact match)
  Edit distance :  0
  CER           :  0.0000  (0 / 7)
============================================================
```

vs. a wrong prediction:

```
============================================================
  FINAL RESULT:  1234568
  (length: 7 digits)
------------------------------------------------------------
  Ground truth  :  1234567
  Sequence acc  :  X ✘ (mismatch)
  Edit distance :  1
  CER           :  0.1429  (1 / 7)
============================================================
```

### What is CER?

**Character Error Rate** = `Levenshtein(pred, gt) / |gt|`.
It's the *minimum* number of single-character edits (insert, delete,
substitute) needed to turn the prediction into the ground truth,
divided by the number of characters in the ground truth. A CER of
0 means perfect; a CER of 1.0 means every character was wrong (or
the prediction was empty).

The Levenshtein implementation lives in `src/ctc/inference.py` and
`src/ctc/train.py` (identical DP, takes two Python lists of ints).

## 3. Step-by-step walkthrough

```python
# src/ctc/inference.py

def predict(image_path, checkpoint_path, visualize=False, ground_truth=None):
    # 1. Load and resize the image to H=64, W=snap-to-multiple-of-8
    img_tensor, orig_img = preprocess_image(image_path)   # [1, 1, 64, W]

    # 2. Build the model and load weights
    model = CRNN_CTC().to(device)
    ckpt  = torch.load(checkpoint_path, map_location=device)
    model.load_state_dict(ckpt['model_state_dict'], strict=False)

    # 3. Forward pass — no second pass for decoding
    model.eval()
    with torch.no_grad():
        logits = model(img_tensor.to(device))    # [1, T, 11]
    decoded = greedy_decode(logits)[0]          # list[int]
    pred_string = "".join(str(d) for d in decoded)

    # 4. (Optional) compute accuracy if GT given
    if ground_truth:
        gt_list   = [int(c) for c in ground_truth if c.isdigit()]
        pred_list = [int(c) for c in pred_string if c.isdigit()]
        seq_match = (gt_list == pred_list)
        edit_dist = _levenshtein(pred_list, gt_list)
        cer       = edit_dist / max(1, len(gt_list))

    # 5. (Optional) show a 2-panel plot
    if visualize:
        ...show image + per-frame argmax trace...

    return pred_string, metrics
```

### The pre-processing step

`preprocess_image`:
1. Opens the image and converts to grayscale.
2. Resizes to **H=64**, with the width scaled to preserve aspect
   ratio.
3. **Snaps the width up to a multiple of 8** — this is essential
   because the 2D CNN downsamples the width by 8× and the 1D stack
   operates on integer time steps.
4. Returns a `[1, 1, 64, W]` tensor (B=1, C=1, H=64, W=variable).

Example: a 200×100 image becomes 64×128 (already a multiple of 8).
The CNN sees it as 16 time steps.

### The forward pass

Exactly the same as the model used during training (just without
`targets` and `target_lengths`):

```python
logits = model(img_tensor)      # [1, T, 11]
```

Returns raw logits. **No log-softmax** — `greedy_decode` only needs
the argmax, which is shift-invariant.

### The greedy decoder

```python
def greedy_decode(logits):
    """logits: [B, T, V] -> list[list[int]]"""
    preds = logits.argmax(dim=-1)        # [B, T]
    blank = config['BLANK_IDX']         # 10
    final = []
    for pred in preds:
        # 1. Collapse consecutive duplicates
        collapsed = [p.item() for i, p in enumerate(pred)
                     if i == 0 or p.item() != pred[i - 1].item()]
        # 2. Remove BLANKs
        cleaned   = [p for p in collapsed if p != blank]
        final.append(cleaned)
    return final
```

Worked example for the frame sequence `1 1 1 1 BLANK 1 1 5`:

- After argmax: `preds = [1, 1, 1, 1, 10, 1, 1, 5]`
- After collapse: `[1, 10, 1, 5]` (four 1s → one 1, two 1s after BLANK → one 1, 5 alone)
- After remove BLANK: `[1, 1, 5]` → `"115"` ✓

## 4. The visualization

When `--visualize` is passed (which the `make ctc-infer` target does
by default), a 2-panel matplotlib figure pops up:

```
┌─────────────────────────────────────┐
│         Input image                 │  ← ax1: original (or aspect-resized) image
│         Predicted: 1234567          │     title shows prediction + (if GT) match
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ 9 │                  ●               │  ← ax2: per-frame argmax trace
│ 8 │            ●                     │     y-axis = predicted class (0-10, 10=BLANK)
│ 7 │      ●         ●                 │     x-axis = CNN time step
│ 6 │    ●                             │
│ 5 │        ●          ●              │     Look for: vertical "spikes" to 10
│ 4 │   ●                              │     (BLANKs) between clusters of digits
│ 3 │ ●   ●     ●                      │     and the cluster itself = a digit
│ 2 │●                                │
│ 1 │                  ●              │
│ 0 │_________________________________│
└─────────────────────────────────────┘
```

### What to look for in the trace

- A **flat horizontal line of 10s** at the right end = padding.
- **A spike to 10** between two clusters = a digit boundary (BLANK).
- **A cluster of N same-value frames** = a digit being predicted for
  N consecutive time steps. A typical digit spans 4-8 frames.
- **A cluster that oscillates** between two values (e.g. 3 ↔ 8) =
  the model is uncertain. This usually resolves correctly with the
  BLANK being emitted in between, but it's a sign the image quality
  is borderline.

If you see the trace going *all the way* to the right edge (no
flat-10 padding) for a long sequence, it means the model's input
width is fully used — i.e. the image had exactly the right width.
That's fine; CTC doesn't require padding.

## 5. Reading the FINAL RESULT banner

```
============================================================
  FINAL RESULT:  1234567
  (length: 7 digits)
------------------------------------------------------------
  Ground truth  :  1234567       ← present only if GT given
  Sequence acc  :  OK ✔ (exact match)
  Edit distance :  0
  CER           :  0.0000  (0 / 7)
============================================================
```

| Field | Meaning |
|-------|---------|
| `FINAL RESULT` | The decoded digit string (no BLANKs, no duplicates) |
| `(length: N digits)` | How many digits the model emitted |
| `Ground truth` | The reference you supplied (or auto-extracted) |
| `Sequence acc` | `OK` (exact match) or `X` (any error) |
| `Edit distance` | The Levenshtein distance between pred and GT |
| `CER` | `edit_distance / len(GT_digits)` |

The banner is **deliberately loud** so it stands out in a long
Colab log even if matplotlib dumps a lot of output above it.

## 6. Makefile command reference

```makefile
# Plain inference
make ctc-infer

# Specific image
make ctc-infer IMAGE=samples/my_image.png

# Explicit ground truth
make ctc-infer GT=1234567

# Auto-extract GT from filename + custom image
make ctc-infer IMAGE=samples/sample_L7_1234567.png

# Skip GT comparison (force)
make ctc-infer NO_GT=1

# Combine
make ctc-infer IMAGE=samples/sample_L20_00112233445566778899.png GT=00112233445566778899
```

You can also call the script directly for finer control:

```bash
python -m src.ctc.inference \
    --checkpoint model/best_ctc.pt \
    --image samples/test.png \
    --ground-truth 12345 \
    --visualize
```

## 7. Batched / directory inference

For evaluating on a directory of images, the simplest path is to
loop in bash:

```bash
for f in samples/ctc_sample_*.png; do
    python -m src.ctc.inference --checkpoint model/best_ctc.pt \
        --image "$f" --no-gt
done
```

For large-scale evaluation with proper seq-acc / CER metrics, use
`src/ctc/evaluate_extrapolation.py` (see
[`CTC_EXTRAPOLATION.md`](./CTC_EXTRAPOLATION.md)) or write a short
custom script that loops over the directory and aggregates metrics.

## 8. Inference gotchas

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| `FileNotFoundError: ...best_ctc.pt` | Wrong checkpoint path | Set `IMAGE` in the Makefile or pass `--checkpoint` directly |
| Empty `FINAL RESULT` (length 0) | Image is all-BLANK, or model is untrained | Check that the model actually trained (val seq-acc > 50%) |
| Predicted length much longer than expected | Model over-emitting, usually from bad input | Verify the image isn't pre-rotated, has visible digits, and is grayscale |
| `FINAL RESULT` has a few extra digits | CTC is over-segmenting (one digit → two digits) | This usually means the width guarantee isn't being respected in your test image; the model is treating padding as digits. Make sure your image is **at least** `num_digits * 16` pixels wide. |
| Banner says "No ground truth available" | Filename doesn't end in digits and you didn't pass `--ground-truth` | Pass `--ground-truth "12345"` explicitly |
